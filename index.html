What is Web Mapping?

Web mapping is the process of using maps delivered by geographic information systems (GIS) through internet browsers. Unlike traditional paper maps, web maps are interactive, allowing users to zoom, pan, search, and access information on demand.

Web mapping is a subset of GIS (Geographic Information Systems), which is a framework for gathering, managing, and analyzing spatial data. While GIS encompasses the entire field of geographic data science, web mapping focuses specifically on delivering map-based information through web browsers.

Modern web maps can display various layers of information, from basic street layouts to complex data visualizations, all accessible through intuitive interfaces.

Components of Web Mapping

Modern web mapping systems consist of several key components working together:


Spatial Database

Stores geographic data including coordinates, shapes, and attributes of map features.

Map Server

Processes requests and delivers map images or data to the client.

Client-Side Library

JavaScript libraries that render maps in the browser and handle user interactions.

Map Layers

Separate visual elements like base maps, points of interest, and data visualizations.

User Interface Controls

Tools for zooming, panning, searching, and interacting with map data.

Geolocation Services

APIs that convert addresses to coordinates and provide routing information.

Spatial Data Formats

Spatial data is the foundation of any web map, containing information about the location and shape of geographic features, along with associated attributes.

GeoJSON

  • Lightweight JSON-based format
  • Natively supported by web browsers
  • Simple structure for points, lines, and polygons
  • Easily parsed with JavaScript
  • Ideal for web applications

Shapefile

  • Traditional GIS format developed by Esri
  • Consists of multiple files (.shp, .dbf, .shx, etc.)
  • Supports complex geometries and attributes
  • Requires conversion for web use
  • Industry standard for desktop GIS
